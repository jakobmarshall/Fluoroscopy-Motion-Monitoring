## How to Contribute

### 1. Reporting Issues

Please open an issue for:
- bugs or unexpected behavior
- documentation gaps
- feature requests
- reproducibility concerns

Include steps to reproduce and relevant environment details when possible.

### 2. Submitting Pull Requests

- Fork the repository and create a feature branch
- Write clear, well-documented code
- Add tests when introducing new functionality
- Ensure your changes do not introduce clinical claims or imply regulatory approval
- Reference related issues in your pull request description

### 3. Data and Privacy

- Do not upload or share any patient-identifiable data
- Use synthetic, anonymized, or publicly available datasets for examples and tests

### 4. Licensing

By submitting a contribution, you agree that your work will be released under the project’s open-source license (Apache 2.0 unless otherwise specified).
